# Environment variables

`rr-server` is configured entirely through environment variables. No configuration file is
required for the server itself — repository-level settings come from each repository's
`.release-regent.toml` file.

## Required variables

These variables must be set. The server will not start without them.

### `GITHUB_APP_ID`

**Type**: integer
**Example**: `123456`

The numeric ID of your GitHub App. Find it at the top of your app's settings page on GitHub.

```bash
GITHUB_APP_ID=123456
```

### `GITHUB_PRIVATE_KEY`

**Type**: string (PEM-encoded RSA private key)

The full contents of the `.pem` file generated by GitHub. Must include the `-----BEGIN RSA PRIVATE KEY-----` and `-----END RSA PRIVATE KEY-----` lines.

```bash
GITHUB_PRIVATE_KEY="-----BEGIN RSA PRIVATE KEY-----
MIIEpAIBAAKCAQEA...
-----END RSA PRIVATE KEY-----"
```

!!! warning "Preserve line breaks"
    The PEM format requires literal newlines. When setting this variable in Docker, Kubernetes,
    or shell scripts, ensure that newlines are preserved. Use `--from-file` in Kubernetes
    secrets and avoid tools that collapse multiline values into a single line. See
    [Deploy the server](../how-to/setup/install-server.md) for guidance.

### `GITHUB_WEBHOOK_SECRET`

**Type**: string
**Example**: `a3f9e2c1d8b74e6f...`

The HMAC-SHA256 secret configured in your GitHub App's webhook settings. Must match exactly.

```bash
GITHUB_WEBHOOK_SECRET=a3f9e2c1d8b74e6f
```

## Optional variables

### `CONFIG_DIR`

**Type**: path
**Default**: current working directory

Directory the server searches for `.release-regent.toml` files.

```bash
CONFIG_DIR=/etc/release-regent/configs
```

### `ALLOWED_REPOS`

**Type**: comma-separated list of `owner/repo` values, or `*`
**Default**: `*` (accept events from any installed repository)

Restricts which repositories the server processes events from. Events from repositories not in
this list are rejected with `403 Forbidden`.

```bash
# Allow two specific repositories
ALLOWED_REPOS=myorg/backend,myorg/frontend

# Allow all repositories (default)
ALLOWED_REPOS=*
```

### `EVENT_CHANNEL_CAPACITY`

**Type**: integer
**Default**: `1024`

Depth of the in-memory bounded channel that buffers incoming webhook events before they are
processed. Under normal load you will never need to change this. Increase it if logs show
"channel full" warnings during bursts of activity.

```bash
EVENT_CHANNEL_CAPACITY=2048
```

!!! note
    This is an in-memory buffer, not a durable queue. Events that overflow the channel are
    dropped. If you expect sustained high throughput, consider whether the default is
    sufficient before increasing this value.

### `PORT`

**Type**: integer
**Default**: `8080`

TCP port the server listens on.

```bash
PORT=9090
```
